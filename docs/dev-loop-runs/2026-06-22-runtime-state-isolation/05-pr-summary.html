Runtime State Isolation

Problem: Evolva runtime artifacts defaulted into the Python package tree, LLM calls had only minimal error recovery, local command execution lacked rollback boundaries, policy behavior was mostly hard-coded, and production safety signals were not all regression-tested.

Change: Runtime state now defaults to .evolva/, with EVOLVA_RUNTIME_HOME for deployment-specific placement. The LLM client now supports runtime timeout config, bounded transient retries, and JSON response validation. Sandbox execution now supports writable roots and failure rollback snapshots. Policy profiles can be configured and audited as JSONL decisions. Observability and Eval now cover policy audit, sandbox rollback, LLM retry, and metric fields. Workflow runs now persist checkpoints and can resume successful unchanged nodes. MCP now has runtime schema cache, health checks, and health/error metrics. Memory and Skills now enforce status/confidence gates before prompt injection. Repo Index now records manifests, detects stale snapshots, and reuses unchanged chunks. Multi-agent collaboration now returns structured reports and emits fallback metrics. Dream/Self-Evolution now stages candidates by default, requires verifier-backed promotion before durable Memory / Skill writes, and exposes the gate/promotion state through CLI/TUI status.

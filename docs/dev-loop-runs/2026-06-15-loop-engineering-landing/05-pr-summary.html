Evolva Loop Engineering 落地增强

本次修改把 Loop 从“可演示的编排概念”推进到“可用于本地生产工程的闭环单元”：可预检、可追踪、可执行真实质量门、可重试、可恢复、可审计，并默认拒绝未显式授权的 shell 命令。

Trace
独立 Loop 自动创建 trace，嵌套 Loop 复用父 trace。
Gates
command_success 通过 Evolva shell 工具实际执行。
Controls
Phase 支持 timeout 与 retries，报告记录每次 attempt。
Validation
新增 CLI/TUI /loop validate、dry-run 与工具/Policy 预检。
Safety
Shell phase 与 command gate 必须显式 command_allowlist。
Resume
run --resume 只复用 phase fingerprint 匹配的成功输出。

Changed Files

  • evolva/loops/spec.py - Loop schema/result metadata, command allowlist, and fingerprint hardening.
  • evolva/loops/runner.py - trace lifecycle, strict validation, allowlisted command gates, retries, dry-run helpers, and resume.
  • evolva/agent/core.py - nested trace-safe chat lifecycle.
  • evolva/cli.py, evolva/tui.py, evolva/loops/__init__.py - command wiring/export.
  • tests/test_loops.py - regression coverage for new loop behavior.
  • README.md - practical Loop Engineering docs and example.

Residual risks and follow-ups
  • 当前适合可信本地生产工程流；多租户或不可信代码执行仍应增加容器/OS 级隔离。
  • 命令 allowlist 是精确/可执行名/前缀匹配，生产环境建议使用窄前缀或完整命令。
  • Retry policy is fixed-count; future versions can add backoff and retry-on predicates.
  • Future versions can add JSON Schema, richer artifact provenance, and containerized execution backends.
